RecordDrive in-flight disclosure revocation evidence
Date: 2026-07-23
Patched project version: RecordDrive 2.0.3

[Confirmed original-code security boundary]
Original route behavior:
- repository download permission checked once by route middleware
- stored file opened through the validated no-follow path
- fs.createReadStream(...).pipe(res) continued without a current-state authorization/session check

Threat scenario:
- an authorized user or stolen authenticated session starts a deliberately slow large download/PDF preview
- the owner/admin revokes download permission, or the session is revoked through logout/password/security change
- the already-open response continues disclosing the file
